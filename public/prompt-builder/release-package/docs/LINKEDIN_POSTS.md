# LinkedIn Launch Copy

## Post 1 — Launch

I built a small AI image tool because I got tired of writing prompts from scratch.

It is called **Prompt Builder**.

The idea is simple:

Most AI image tools ask you to “describe your image.”

But film people do not think that way. They think in shot size, lens, camera angle, lighting setup, aspect ratio, mood, and reference.

So I built an interface where you can select things like:

- epic wide shot
- 35mm anamorphic
- blue-hour industrial glow
- practical fill
- teal-and-amber color grade
- man small in frame for scale

Then the tool turns that into a prompt formatted for Midjourney, Flux, Nano Banana, or GPT Image.

No account. No backend. Installable as a PWA. For GitHub users, it can also launch from a one-command CLI path. LLM enhancement is optional, but the core tool works without it.

This is the product thesis I care about:

> AI tools get better when they speak the language of the user’s craft.

Prompt Builder is my first expression of that idea.

## Post 2 — Product Thinking

I do not think the future of AI tools is just “better models.”

I think a huge part of the next wave is better input layers.

Prompt Builder started from a very specific frustration:

AI image models are powerful, but the user has to translate visual intent into prompt syntax.

For film and visual creators, that translation is weird. They already have a precise language:

- shot size
- focal length
- lighting motivation
- aspect ratio
- production design
- mood
- continuity
- visual reference

So I asked: what if the interface used that language first?

The result is a prompt builder where users select production terms and the tool maps them into model-specific outputs.

Midjourney gets `/imagine`.
Flux gets natural language.
Nano Banana gets conversational direction.
GPT Image gets structured JSON.

Same intent, different model dialects.

That is the part I find interesting: not prompt engineering as a trick, but domain expertise as product design.

## Post 3 — HR / Hiring Audience

I shipped a portfolio project that combines my film-production background with AI product design.

It is called **Prompt Builder**: an installable web app that helps users build AI image prompts through visual presets, chips, and model-specific formatting.

What it demonstrates:

- PWA publishing

This is the kind of AI tooling I want to build more of: practical, domain-specific, and grounded in how people actually work.

## Post 4 — Investor / Product Strategy Angle

One product idea I keep coming back to:

AI-native tools should not start from blank text boxes.

Blank text boxes are flexible, but they push too much translation work onto the user.

Prompt Builder is a small experiment in the opposite direction.

It asks: what if expert workflow knowledge became the interface?

For film and visual production, that means:

- shot language
- lens language
- lighting setups
- continuity rules
- production categories
- model-specific output formats

The tool is not trying to replace creativity. It is trying to remove the syntax tax between intent and output.

That pattern feels bigger than image generation.

Every domain has its own language. The opportunity is turning that language into useful AI interfaces.
